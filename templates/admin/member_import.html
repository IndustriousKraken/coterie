Bulk Import Members

Upload a CSV file to create multiple members in one operation. Per-row errors (bad email, duplicate, unknown membership type) are reported individually — they don't abort the rest of the batch.

Maximum 5 MB.

CSV format reminder

Required columns (any order, case-insensitive): email, username, full_name, membership_type_slug.

Optional columns: status (defaults to Pending), notes, discord_id.

Other columns from a previous export (id, joined_at, is_admin, bypass_dues, email_verified_at, dues_paid_until) are silently ignored — the system controls those fields.

Imported members start Pending (or the row's specified status) with no password. Members will need to use password-reset to claim their account.

Example (one data row):

email,username,full_name,membership_type_slug,status,notes
jane@example.com,jdoe,Jane Doe,regular,Pending,Imported 2026-05
