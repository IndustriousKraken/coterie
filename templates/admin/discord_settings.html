1. Create a bot

Go to discord.com/developers/applications, click "New Application", give it a name like "Coterie", then in the "Bot" tab click "Reset Token" to reveal the bot token. Paste that above.

2. Invite the bot

In the same developer portal, "OAuth2 → URL Generator", check bot, and under permissions check Manage Roles and Send Messages. Open the generated URL in a new tab and add the bot to your server.

3. Create roles

In your Discord server, create two roles — e.g. Coterie Member and Coterie Expired. Position both BELOW the bot's own role in the role list, otherwise Discord won't let the bot assign them.

4. Get the IDs

Enable Developer Mode in your Discord User Settings → Advanced. Then right-click anything to "Copy ID":

  • Server name → Guild ID
  • Each role → Role ID
  • Each channel → Channel ID

5. Channel permissions ("the jail")

Coterie just toggles roles. Set up which channels each role can see in your Discord server's channel-level permissions: the Member role should see most channels, the Expired role should typically only see one "please update your dues" channel where moderators can verify and remove the role manually (or members can pay through the portal, which removes the role automatically on next sync).

Common errors

401 Unauthorized
Bot token wrong, missing, or revoked. Reset in dev portal and paste again.
403 Forbidden — bot is missing the required permission
Either the bot doesn't have Manage Roles, OR its role is positioned below the role it's trying to assign. Move the Coterie bot's role above Member/Expired in your role list.
404 Not Found
Snowflake ID is for a deleted role/channel/guild, or the user isn't in this guild. The integration logs and continues.
