## 1. Add factory-branch tests for `bot_challenge::from_config`

- [ ] 1.1 `from_config_returns_disabled_verifier_when_provider_disabled` — build `BotChallengeConfig { provider: "disabled".into(), ..Default::default() }`, call `from_config(&cfg, reqwest::Client::new())`, assert `verify("public/signup", None, None).await` returns `Ok(())`. Probe with `verify("public/signup", Some("anything"), None).await` and assert `Ok(())` too — distinguishes `DisabledVerifier` from `TurnstileVerifier` (which would issue `Err(Missing)` then a real HTTP call).
- [ ] 1.2 `from_config_falls_back_to_disabled_when_turnstile_secret_empty` — build `BotChallengeConfig { provider: "turnstile".into(), secret_key: String::new(), ..Default::default() }`, assert that `verify(None, ...)` returns `Ok(())` (confirms fall-back to `DisabledVerifier`, not construction of a misconfigured `TurnstileVerifier`).
- [ ] 1.3 `from_config_falls_back_to_disabled_for_unknown_provider` — build `BotChallengeConfig { provider: "nonsense-provider".into(), secret_key: "anything".into(), ..Default::default() }`, assert that `verify(None, ...)` returns `Ok(())`.
- [ ] 1.4 `from_config_returns_turnstile_verifier_when_configured` — build `BotChallengeConfig { provider: "turnstile".into(), secret_key: "sk_test".into(), verification_url: "http://127.0.0.1:1/never".into(), timeout_ms: 50, ..Default::default() }`, assert that `verify("r", None, None).await` returns `Err(VerifyError::Missing)` — proves the returned arc is a real `TurnstileVerifier` (rejects missing token without I/O).
- [ ] 1.5 `from_config_returns_turnstile_verifier_when_hcaptcha_configured` — same as 1.4 but with `provider: "hcaptcha".into()` to assert that hcaptcha also takes the `TurnstileVerifier` path (the factory groups them together).
